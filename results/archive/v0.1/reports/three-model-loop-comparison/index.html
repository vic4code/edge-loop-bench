LOOP DESIGN

Controller flow

The nodes below match the tested controller

Direct baseline

One attempt; no controller retry
  1. Reset clean task worktree
  2. One model call returns replacement-edit JSON
  3. Validate paths and apply candidate
  4. Run public tests once
  5. If public tests pass, run isolated hidden evaluation

Bounded Retry

Repair loop within shared logical caps
  1. Build prompt from the current worktree
  2. Model returns replacement-edit JSON
  3. Validate, apply, and run public tests
  4. On rejection or failure, add sanitized feedback
  5. Repeat until pass or call/token/test budget ends
  6. If public tests pass, run isolated hidden evaluation

Maker–Verifier

Tested implementation is review-and-revise
  1. Maker returns replacement-edit JSON
  2. Validate, apply, and run public tests
  3. Request a second model call as verifier
  4. Review and revise: verifier may replace source files
  5. Apply revision and run public tests again
  6. If public tests pass, run isolated hidden evaluation
Evaluation boundaryHidden evaluator feedback is never returned to any model. Maker–Verifier here is not an independent read-only APPROVE/REJECT judge; it is a second review-and-revise edit call.
