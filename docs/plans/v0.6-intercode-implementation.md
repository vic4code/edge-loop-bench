# v0.6 InterCode-Bash implementation and execution plan

Status: **active**

Design authority: [`experiment-design-v0.6-intercode-bash.md`](../experiment-design-v0.6-intercode-bash.md)

Decision: [`ADR 012`](../decisions/012-intercode-bash-causal-loop.md)

## Phase 0 — Freeze sources and semantics

- Record the official InterCode commit and every source-file SHA-256.
- Vendor only required Bash data, setup assets, license, and attribution.
- Define stable task IDs and separate the 24 calibration rows from the
  200-row qualification source population.
- Freeze the three-channel information boundary and four arm semantics.
- Commit this phase before controller implementation.

Exit: documentation states what can and cannot be claimed; no measured scoring
is permitted.

## Phase 1 — Test-first interactive core

Add failing tests for:

- interactive manifests and rejection of mixed legacy/interactive arms;
- one shared candidate-1 prompt hash and seed across all arms;
- Independent contexts containing no previous action, observation, or score;
- Raw receiving only the frozen output/reward packet;
- Engineered receiving structured evidence only after candidate 1;
- deterministic rollback and repeated-no-progress stopping;
- strict separation of observation, stop signal, and final evaluator output;
- exact logical-token, prompt-class, action, evaluator, and reset accounting;
- append-only resume and unchanged legacy execution.

Implement a fake in-memory environment first. No Docker behavior is required to
prove the controller state machine.

Exit: focused tests pass and legacy result fixtures remain loadable.

## Phase 2 — Pinned offline Bash adapter

- Add an EdgeLoop-owned environment adapter instead of importing upstream
  experiment scripts at runtime.
- Build four native-arm64 images from immutable base digests.
- Enforce unique containers, `--network none`, no host mounts/socket, resource
  limits, capability drops, and cleanup verification.
- Quarantine all upstream evaluator `info`.
- Add deterministic stdout bounds, action parsing, timeouts, state digests,
  checkpoints, and restore.
- Implement repeat-twice gold qualification and ordered exclusion records.
- Implement and adversarially test the strict final evaluator.

Exit: all qualified tasks replay identically in fresh containers, reset probes
show no cross-episode state, and the suite manifest is committed.

## Phase 3 — Accounting and reporting

- Add interactive environment identity to manifest compatibility signatures.
- Record append-only events for resets, model calls, actions, stop signals,
  checkpoints, restores, stop decisions, strict evaluation, and cleanup.
- Count logical repeated prompt tokens even under prefix caching.
- Separate main prompts, automatic feedback follow-ups, and independent sample
  prompts; never infer human prompts.
- Add attempt- and token-budget curves, strict/official endpoints, paired
  transitions, invalidity, and per-stratum tables to HTML and Markdown output.
- Keep one canonical `results/OPEN-ME/index.html` entry point only after a valid
  confirmatory result exists.

Exit: `make check` passes and a leak-focused review finds no gold, evaluator
path, secret, unstable timestamp, or unsupported claim in publishable output.

## Phase 4 — Safety and one-task smoke

- Audit Docker daemon behavior and abort with a report if unrelated containers
  are running. Never stop or mutate them automatically; measured execution
  starts only after their owner has stopped them externally.
- Prove reset/isolation with a state-mutating task.
- Verify candidate-1 request hashes across arms with a fake model, then Qwen.
- Run one calibration task through four arms with Qwen3.5 4B.
- Freeze prompts, `K = 10`, the deterministic `B*` calculation, all numeric
  gates, and admission/cooldown thresholds before the first calibration model
  call; then run the 30-minute combined model-plus-Docker host qualification.

Exit: no memory-pressure, swap-growth, thermal, isolation, or accounting gate
fails. A smoke result is engineering evidence only.

## Phase 5 — Disjoint calibration

Run all 24 quickstart calibration tasks, sequentially and append-only, on:

1. Qwen3.5 4B;
2. Phi-4-mini only after Qwen host safety is established.

Verify parser/admissibility, floor/ceiling, seed diversity, context headroom,
action timeout, and score/strict disagreement against the already frozen gate
manifest. Calibration can accept or reject the model/design; it cannot tune
budgets, prompts, `K`, controller/evaluator hashes, image digests, task manifest,
or block order after outputs are viewed.

Exit: each admitted model has a signed-off calibration record. Failed models
are explicitly excluded; no confirmatory task outputs have been opened.

## Phase 6 — Confirmatory execution

- Execute Qwen first, one four-arm block at a time.
- On reboot, resume from the first missing valid block.
- On a resource breach, invalidate and requeue the complete four-arm block, at
  most twice.
- Execute Phi as a separate replication only if its calibration passes.
- Do not inspect aggregate arm differences until the declared run is complete,
  except safety, coverage, and infrastructure-validity dashboards.

Exit: complete manifest-bound coverage or a documented safety stop.

## Phase 7 — Analysis and publication

- Open the frozen aggregate and compute task-clustered paired bootstrap
  intervals, transitions, token curves, prompt counts, and cost per net rescue.
- Report filesystem strata and models separately before any pooled view.
- State whether Raw beats verified sampling and whether Engineered beats Raw.
- Distinguish inconclusive, interaction-only, engineered-loop-positive, and
  model-task-floor outcomes.
- Put complete aggregate data and reproducibility identities in README; keep raw
  append-only events and derived artifacts in their documented locations.
- Update the canonical HTML only for a complete valid result, then draft the
  LinkedIn summary from the same numbers.

Exit: every public number is derivable from committed raw records and the final
claim matches the preregistered decision rules.
