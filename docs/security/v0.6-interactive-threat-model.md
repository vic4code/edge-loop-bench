# v0.6 interactive benchmark threat model

- Status: **required implementation gate**
- Scope: `InterCode-Bash-qualified@c3e46d8`
- Reviewed: **2026-07-15**

This document covers benchmark integrity and host isolation. It does not claim
that Docker is a security boundary against every kernel or daemon compromise.
The measured study remains blocked until every P0 control below has an automated
test against the exact local Docker version and immutable image identities.

## Trust boundary

```text
model bytes -> action parser -> controller -> Docker daemon -> agent container
                                      |                    |
                                      |                    v
                                      |              private checkpoint
                                      v                    |
                               trusted evaluator <---------+
                                      |
                                      v
                            typed score / strict boolean

private hash-chained events -> allowlisted derivation -> README / HTML / JSON
```

The model, action bytes, agent-container filesystem, stdout/stderr, and every
upstream evaluator detail are untrusted. The controller, frozen adapter, clean
image, external gold baseline, evaluator container, checkpoint store, private
journal, and allowlisted publisher are trusted only after identity validation.

Protected assets are the host and Docker daemon, gold/evaluator secrecy,
checkpoint identity, score correctness, event provenance, task isolation, and
the distinction between private telemetry and publishable results.

## STRIDE register

| ID | Class | Failure or abuse case | Required control | Gate |
| --- | --- | --- | --- | --- |
| S1 | Spoofing | Model output pretends to be a score, stop signal, or journal event | Strict action JSON; typed evaluator return; raw model bytes never merged as event fields | Fake core |
| S2 | Spoofing | A checkpoint from another run, arm, task, image, or attempt is restored or scored | Opaque handle bound to all identities plus state digest; reject stale or cross-scope handles | Fake + Docker |
| T1 | Tampering / elevation | Quotes, backticks, `$()`, separators, newlines, or leading options escape to a host shell | Never use `shell=True`; fixed Docker argv; command is one container-side argv value; adversarial host-marker test | Docker P0 |
| T2 | Tampering | Agent replaces `git`, checksum tools, `PATH`, `.git`, or evaluator helpers | Evaluator uses a clean image, trusted binaries, external baseline, and never imports executable state from the agent layer | Docker P0 |
| T3 | Tampering | A background process mutates state between action, checkpoint, and score | Process-delta audit, freeze/stop before snapshot, content digest binding, destroy contaminated container | Docker P0 |
| T4 | Tampering / repudiation | Journal is edited, deleted, reordered, duplicated, or partially written | Canonical JSON, contiguous sequence, previous-event hash, event hash, explicit terminal seal, explicit partial-tail recovery | Fake core |
| R1 | Repudiation | Crash after a side effect causes silent replay | Durable intent event before model/container/action/checkpoint/evaluator/restore/strict calls; unsealed episode is invalid and cannot resume in place | Fake core |
| I1 | Disclosure | Gold command, evaluator path, reward details, strict diff, or checkpoint handle enters a prompt or report | Three typed channels, evaluator canaries, recursive public allowlist, private blobs referenced only by digest | Fake + Docker |
| I2 | Disclosure | `/proc`, environment, control sequences, bidirectional text, or host-looking paths leak through output | Minimal env, private bounded capture, normalized model observation, escaped allowlisted publication | Docker + publisher P0 |
| D1 | Denial of service | Fork, sleep, memory pressure, unbounded output, or background daemon survives timeout | PID/CPU/memory limits, action deadline, streaming output cap, and container destruction on breach | Docker P0 |
| D2 | Denial of service | Disk bomb or repeated snapshots exhaust Docker Desktop storage | Writable-byte/file-count/checkpoint ceilings, free-space admission, run-scoped cleanup | Docker P0 |
| D3 | Denial of service | Malicious tree causes evaluator recursion, symlink loops, or path explosion | File/depth/byte limits, no symlink following, evaluator deadline and resource limits | Docker P0 |
| E1 | Elevation | Agent reaches Docker socket, host mounts, devices, daemon API, or another container | No mounts/socket/devices, `--network none`, non-privileged UID, dropped capabilities, no-new-privileges, seccomp, private PID/IPC | Docker P0 |
| E2 | Elevation | Snapshot/archive contains absolute paths, `..`, unsafe links, or device nodes | Rooted safe reader; reject traversal, special files, and links escaping the snapshot root | Docker P0 |

## Mandatory fake-core evidence

The fake core must prove all of the following before Docker adapter code can be
used for a measured episode:

- candidate-one prompt bytes and seed are identical across all four arms;
- Independent receives no prior action, output, score, or failure signal;
- Raw receives only the frozen observation and scalar-score feedback packet;
- Engineered treatment begins after candidate one and has deterministic
  rollback, tie selection, and no-progress behavior;
- parser failures consume model/token budget but no environment, checkpoint, or
  evaluator budget;
- model output cannot inject a reward, success bit, journal type, or duplicate
  JSON key;
- action parsing rejects oversize, invalid Unicode, control, and line-separator
  ambiguity while leaving shell metacharacters inert for the adapter;
- evaluator types have no diagnostic or path field;
- every external side effect has a prior durable intent record;
- a crash at every side-effect boundary leaves a valid unsealed journal and
  the same log cannot be executed again;
- a successful episode has one verifiable terminal seal.

## Mandatory Docker-adapter evidence

Measured qualification remains blocked until automated tests establish:

1. Exact inspected security profile: immutable image digest, `NetworkMode=none`,
   empty mounts/devices, no socket, `Privileged=false`, dropped capabilities,
   no-new-privileges, seccomp, private PID/IPC, frozen UID, and explicit
   CPU/memory/PID limits.
2. Adversarial commands cannot create a host marker and can affect only their
   own isolated agent container.
3. Timeout, output, process, memory, and disk breaches destroy only resources
   labeled with the exact v0.6 run identity. Unrelated containers are never
   stopped or changed.
4. Checkpoints are atomic, content-addressed, size-bounded, preserve the
   declared writable state and working directory, and reject unsafe archives.
5. Fresh episodes cannot observe files, environment, processes, cwd state, or
   metadata from an earlier task, arm, or seed.
6. Agent-modified tools and `.git` cannot influence either official or strict
   evaluation; scores bind to the exact checkpoint/action/output/state tuple.
7. Gold, evaluator-path, strict-diff, and container-identity canaries are absent
   from every later model request and every public artifact.
8. Docker admission rejects an unexpected context, remote daemon, unrelated
   running container, insufficient free space, or an image/tag whose resolved
   digest differs from the frozen manifest.

Docker's own documentation notes that resource constraints must be configured,
bind mounts can modify the host, Docker-daemon control is highly privileged,
the `none` network driver retains only loopback, and `docker commit` excludes
mounted volume data. The adapter therefore verifies its actual inspected state
instead of relying on defaults.

## Publication boundary

Public records use an allowlist, not a blacklist. The default public surface is
limited to stable task/model/controller identifiers, hashes, bounded enums,
numeric counters, official score, strict boolean, relative durations, coverage,
and exclusion reason codes.

Raw commands, model prose, complete stdout/stderr, checkpoint handles, host or
container paths/names, evaluator `info`, gold commands, strict diagnostics,
environment dumps, and Docker inspection payloads stay private. Publishing
trajectories would require a separate redaction pipeline and canary suite; it
cannot be enabled by a flag on the normal result serializer.
